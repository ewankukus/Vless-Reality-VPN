# VLESS + Reality VPN Server

Скрипт автоматической установки и оптимизации VPN-сервера на базе 3X-UI + Xray-core с протоколом VLESS + Reality.

## Что делает скрипт

- Устанавливает панель управления [3X-UI](https://github.com/mhsanaei/3x-ui) v2.8.11+
- Настраивает SSL-сертификат Let's Encrypt (авто-обновление)
- Применяет 14 оптимизаций производительности и безопасности
- Настраивает файрвол (UFW), fail2ban, NTP, swap

## Минимальные требования к серверу

| Параметр | Минимум | Рекомендуется |
|----------|---------|---------------|
| ОС | Ubuntu 22.04 LTS | Ubuntu 24.04 LTS |
| RAM | 1 ГБ | 1 ГБ (скрипт сам создаёт swap) |
| CPU | 1 ядро | 1 ядро |
| Сеть | 100 Мбит/с | 1 Гбит/с |
| Трафик | от 500 ГБ/мес | 1 ТБ/мес |
| Доступ | Root SSH | Root SSH + ключ |
| Расположение | За пределами РФ | UK / NL / DE |

> Для 5-20 пользователей достаточно самого дешёвого VPS за ~$5/мес.

## Подготовка VPS (перед установкой)

При заказе/переустановке VPS выберите:

| Настройка | Что выбрать | Почему |
|-----------|-------------|--------|
| **ОС** | Ubuntu 24.04 LTS Minimal | Легковесная, LTS до 2029 |
| **Swap** | None | Скрипт сам создаёт оптимальный swap 512MB |
| **SSH-ключ** | Не выбирать | Пароль придёт на почту после установки |
| **Hostname** | Любое (например `vpn`) | Не влияет на работу |

### Подключение к серверу

После создания VPS провайдер пришлёт на почту: **IP-адрес**, **логин** (root) и **пароль**.

```bash
ssh root@IP_СЕРВЕРА
```

Введите пароль из письма. После входа запустите скрипт установки.

## Установка

Подключитесь к серверу по SSH и выполните одну команду:

```bash
bash <(curl -sL https://raw.githubusercontent.com/ewankukus/Vless-Reality-VPN/main/install-vpn.sh)
```

## Настройка VLESS + Reality

### 1. Откройте панель

Перейдите по URL из вывода установки:
```
https://IP_СЕРВЕРА:ПОРТ/БАЗОВЫЙ_ПУТЬ/
```

### 2. Создайте Inbound

**Панель → Inbounds → Add Inbound**

| Поле | Значение |
|------|----------|
| Примечание | `vless-reality` |
| Протокол | `vless` |
| Порт | `443` |
| Транспорт | `TCP (RAW)` |
| Безопасность | `Reality` |
| uTLS | `chrome` |
| Target | `www.samsung.com:443` |
| SNI | `www.samsung.com` |
| SpiderX | `/` |

Нажмите **Get New Keys** для генерации ключей Reality.

### 3. Настройте клиента

| Поле | Значение |
|------|----------|
| Email | любое имя (например `user1`) |
| Flow | `xtls-rprx-vision` **(обязательно!)** |

### 4. Sniffing

| Поле | Значение |
|------|----------|
| Включено | ВКЛ |
| HTTP | ВКЛ |
| TLS | ВКЛ |
| QUIC | ВКЛ |
| FAKEDNS | ВКЛ |
| Metadata Only | ВЫКЛ |
| Route Only | ВЫКЛ |

Нажмите **Создать**.

### 5. Получите ссылку подключения

1. В списке Inbounds нажмите **≡** рядом с инбаундом
2. Нажмите иконку **QR-код** или **инфо** у клиента
3. Скопируйте ссылку `vless://...` или отсканируйте QR

## Альтернативные Reality-цели

Если `www.samsung.com` заблокируют, смените Target/SNI на:

| Цель | Описание |
|------|----------|
| `www.mozilla.org` | Mozilla, TLS 1.3, H2 |
| `www.asus.com` | Низкий профиль, TLS 1.3 |
| `dl.google.com` | CDN загрузок Google |
| `www.logitech.com` | Производитель периферии |

### Для белых списков РФ операторов

Когда работают только одобренные домены:

| Цель | Почему |
|------|--------|
| `tunnel.vk.com` | Инфраструктура VK, всегда в белом списке |
| `ps.userapi.com` | CDN VK |
| `yandex.ru` | Ядро Яндекса, никогда не блокируется |
| `www.sberbank.ru` | Банковский сервис, всегда доступен |

## Оптимизации

Скрипт автоматически применяет:

| Оптимизация | Значение | Зачем |
|-------------|----------|-------|
| BBR | `fq` + `bbr` | Лучший алгоритм для дальних соединений |
| TCP буферы | 4MB макс. | Оптимально для 1GB RAM |
| tcp_notsent_lowat | 131072 | Снижает задержку для прокси |
| TCP FastOpen | 3 (клиент+сервер) | Быстрее установка соединения |
| MTU Probing | включен | Избежание фрагментации пакетов |
| Порты | 1024-65535 | Больше исходящих соединений |
| Swap | 512MB | Защита от OOM на серверах 1GB |
| fail2ban | SSH, 5 попыток, бан 1ч | Защита от брутфорса |
| Chrony/NTP | авто | Критично для TLS/Reality |
| UFW | deny all + allow list | Минимальная поверхность атаки |
| nofile | 51200 | Достаточно для тысяч соединений |
| Автообновления | только безопасность | Авто-патчинг уязвимостей |
| Защита ядра | rp_filter, no redirects | Стандартное усиление |

## Переменные конфигурации

Можно переопределить при запуске:

```bash
PANEL_PORT=9999 REALITY_TARGET=www.mozilla.org bash install-vpn.sh
```

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `PANEL_PORT` | 2053 | Порт панели 3X-UI |
| `SUB_PORT` | 2096 | Порт сервера подписок |
| `VLESS_PORT` | 443 | Порт VLESS Reality |
| `SWAP_MB` | 512 | Размер swap в МБ |
| `REALITY_TARGET` | www.samsung.com | Домен маскировки Reality |

## Безопасность после установки

### 1. Сменить пароль root

```bash
passwd
```

### 2. Отключить вход по паролю SSH

> Только после настройки SSH-ключей!

```bash
sed -i 's/#\?PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config
sed -i 's/#\?PermitRootLogin.*/PermitRootLogin prohibit-password/' /etc/ssh/sshd_config
systemctl restart sshd
```

### 3. Ограничить доступ к панели

```bash
ufw delete allow ПОРТ_ПАНЕЛИ/tcp
ufw allow from ВАШ_IP to any port ПОРТ_ПАНЕЛИ proto tcp
```

## Решение проблем

### SSL-сертификат не выдаётся

```bash
# Проверить что порт 80 открыт
ufw status | grep 80

# Перевыпустить вручную
~/.acme.sh/acme.sh --issue -d IP_СЕРВЕРА --standalone --listen-v4
```

### Панель недоступна

```bash
systemctl status x-ui        # Статус сервиса
ss -tlnp | grep ПОРТ         # Проверка порта
ufw status | grep ПОРТ       # Проверка файрвола
systemctl restart x-ui        # Перезапуск
```

### Низкая скорость

```bash
# Проверить BBR
sysctl net.ipv4.tcp_congestion_control  # Должно быть: bbr

# Нагрузка
top -bn1 | head -5
free -h
```
